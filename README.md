#  StarSnap AR – Desktop Application

StarSnap AR is a **desktop application built with Electron, React, Vite, and TailwindCSS** that provides an interactive UI with camera/microphone support and cloud-based media storage.
The application runs as a **native desktop app** while using modern web technologies for development.

The project integrates **Cloudinary** for media storage and delivery.

---

#  Project Overview

StarSnap AR runs a React frontend inside an Electron shell.

Architecture overview:

```
Electron Main Process
        │
        │  (Loads UI)
        ▼
React + Vite Frontend
        │
        │ (IPC communication)
        ▼
Preload Script (contextBridge)
        │
        │ (Image Upload)
        ▼
Cloudinary Cloud Storage
```

Key components:

| Component        | Purpose                    |
| ---------------- | -------------------------- |
| Electron         | Desktop runtime            |
| React            | UI framework               |
| Vite             | Build tool and dev server  |
| TailwindCSS      | Styling framework          |
| Electron Builder | Creates desktop installers |
| Cloudinary       | Cloud media storage        |

#  Electron Security Architecture

Electron applications have **two environments**.

---

## Main Process

Runs Node.js and controls the desktop application.

File:

```
main.js
```

Responsibilities:

* Creating windows
* Handling application lifecycle
* Managing permissions
* Registering custom protocols

---

## Renderer Process

Runs the React frontend.

Security configuration:

```
nodeIntegration: false
contextIsolation: true
```

This prevents direct Node.js access from the UI.

---

## Preload Script

The preload script acts as a **secure bridge**.

File:

```
preload.js
```

Example API exposed:

```
window.electron.sendData(data)
```

This allows safe communication between Electron and React.

---

#  Custom Protocol

The app uses a custom protocol:

```
app://
```

Example:

```
app://index.html
```

This allows Electron to securely load files from the `dist` folder.

#  Environment Variables

Create a `.env` file in the root project directory.

Example:

```
VITE_CLOUDINARY_CLOUD_NAME=your_cloud_name
VITE_CLOUDINARY_UPLOAD_PRESET=your_upload_preset
```

These variables allow the frontend to upload images directly to Cloudinary.

---

# 📤 Cloudinary Upload Workflow

```
User captures image
        │
        ▼
React frontend prepares upload request
        │
        ▼
Cloudinary Upload API
        │
        ▼
Cloudinary stores the image
        │
        ▼
Secure image URL returned
        │
        ▼
Application displays or stores the URL
```

Upload endpoint example:

```
https://api.cloudinary.com/v1_1/<cloud_name>/image/upload
```

POST request fields:

* file
* upload_preset

---

# Cloudinary Security Notes

For production applications:

* Never expose **API Secret**
* Use **upload presets**
* Use a backend for signed uploads if needed
